# Feature Specification: 角色头像与旁白输入

**Feature Branch**: `002-avatar-and-narration`

**Created**: 2026-06-11

**Status**: Draft

**Input**: User description: "①角色设定中增加头像（上传或 Imagine Provider 生成）②对话中持续显示角色头像 ③输入框上方增加非必填旁白输入框，填写后与用户消息一起传入，提示词区分用户输入与旁白。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 角色头像 (Priority: P1)

用户在角色编辑界面为角色设置头像：从本地选一张图片上传，或一键用 Imagine Provider
基于该角色的外貌设定生成一张；之后在聊天界面，角色的每条消息旁都显示该头像。

**Why this priority**: 直接提升对话沉浸感，且头像展示依赖头像数据，两者一体交付。

**Independent Test**: 为角色上传/生成头像后进入对话，验证角色消息旁持续显示头像；
更换与移除头像后展示同步更新。

**Acceptance Scenarios**:

1. **Given** 角色编辑界面，**When** 用户选择本地图片，**Then** 头像立即预览并随角色保存（文件落本地，重启可见）。
2. **Given** 已配置 Imagine Provider 且角色填写了外貌，**When** 点「生成头像」，**Then** 后台生成并落盘为头像；失败时给出轻量提示、不影响其他编辑。
3. **Given** 角色有头像，**When** 进入任意对话，**Then** 每条角色消息旁显示圆形头像；无头像时显示默认占位图标。
4. **Given** 角色编辑界面，**When** 移除头像，**Then** 本地文件删除，对话中回退占位图标。

---

### User Story 2 - 旁白输入 (Priority: P2)

用户在聊天输入框上方的旁白框里描述场景或情节（如「夜晚，两人走在江边」），再输入
消息发送；模型同时收到旁白与发言并能区分两者；旁白在对话流中以特殊样式展示。

**Why this priority**: 增强剧情演出能力；独立于头像，可单独验证。

**Independent Test**: 填旁白+发言发送，观察角色回复明显受场景影响；不填旁白时行为与
现状完全一致。

**Acceptance Scenarios**:

1. **Given** 旁白框为空，**When** 发送消息，**Then** 与现有行为完全一致（提示词不含任何旁白标记）。
2. **Given** 旁白框有内容，**When** 发送，**Then** 发给模型的该条用户消息中旁白与用户发言以明确标记区分；旁白框随后自动清空。
3. **Given** 含旁白的历史消息，**When** 查看对话流或重启应用，**Then** 旁白以区别于正文的样式显示在该条消息中，且持久保留。
4. **Given** 含旁白的消息处于记忆窗口内，**When** 继续对话，**Then** 上下文中该消息仍携带旁白标记（含重新生成场景）。

---

### Edge Cases

- 仅填旁白、未填发言 → 发送按钮不可用（旁白非独立消息）。
- 生成头像时无 Imagine Provider / 外貌为空 → 入口禁用并说明原因。
- 删除角色 → 头像文件一并清理。
- 头像图片文件丢失 → 回退占位图标，不报错。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-101**: 角色 MUST 支持头像，来源两种：本地上传、Imagine Provider 基于外貌设定生成；头像文件 MUST 存本地，消息中只引用文件名。
- **FR-102**: 头像 MUST 可更换、可移除；移除/换角色头像/删除角色时 MUST 清理旧文件。
- **FR-103**: 对话流中每条角色消息旁 MUST 持续显示头像；无头像时显示占位。
- **FR-104**: 聊天输入框上方 MUST 有非必填旁白输入框；为空时发送行为与无此功能完全一致。
- **FR-105**: 旁白非空时 MUST 与用户发言合并为一条用户消息发送，提示词内以固定标记区分「旁白」与「用户发言」；标记格式硬编码（宪法 VI）。
- **FR-106**: 旁白 MUST 随消息持久化，在对话流中以区别样式展示，并保持在上下文窗口与压缩输入中。

## Success Criteria *(mandatory)*

- **SC-101**: 头像设置后 100% 的角色消息旁可见头像，重启不丢失。
- **SC-102**: 不使用旁白的用户其请求体与本迭代前逐字节一致（零回归）。
- **SC-103**: 头像生成失败不阻塞角色保存与对话。

## Assumptions

- 头像与生成图片共用本地图片存储目录。
- 头像生成使用全局任选一个 Imagine Provider（编辑界面内选择）。
- 旁白只附属于用户消息，不能单独发送。
