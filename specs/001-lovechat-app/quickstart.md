# Quickstart: LoveChat 验证指南

**Plan**: [plan.md](./plan.md) | 契约见 [contracts/](./contracts/)

## 前置条件

- macOS 26 (Tahoe)+，安装配套 Xcode
- 至少一个可用的 OpenAI 或 Anthropic 标准 API 服务（Base URL + API Key + 模型名）
- （可选）一个 OpenAI 标准 Images API 服务

## 编译运行

1. `open LoveChat.xcodeproj`
2. 选择 My Mac，⌘R 运行（首次需在 Signing 里选本机团队/Sign to Run Locally）

## 验证场景（对应 spec.md 用户故事）

### V1 — 配置与首次对话（US1）

1. ⌘, 打开设置 → Provider → 添加 Chat Provider（四项必填 + 协议类型）→ 保存
2. 点「测试连接」→ 应显示成功（或带原因的失败）
3. 设置 → 角色 → 新建角色（只填名称）→ 保存
4. 主窗口新建对话，选择角色与 Provider，发送一条消息
5. **预期**: 回复流式逐字出现；期间「停止」按钮可用且点击即停
6. **密钥安全检查**: `grep -r "<你的key前8位>" ~/Library/Containers/<bundle-id>/` 无命中

### V2 — 角色与心理活动（US2）

1. 角色开启「展示心理活动」，对话
2. **预期**: 括号内容以特殊样式（如斜体淡色气泡）渲染；故意诱导模型输出不配对
   括号时内容按普通文本显示、不丢字

### V3 — 对话管理（US3）

1. 进行 2+ 个对话后重启 App → 历史卡片完整
2. 选择旧对话继续聊 → 上下文延续
3. 导出 Markdown → 文件含完整对话、不含任何密钥/配置
4. 重新生成最后回复、编辑一条消息、删除一条消息、删除整个对话 → 均立即生效并持久化

### V4 — 记忆与压缩（US4）

1. 设置中将记忆轮数调为 1（验证下限钳制：尝试 0 与 21 应被拒绝）
2. 压缩阈值尝试设为大于记忆轮数 → 界面强制钳回
3. 聊超过阈值轮数，在早期透露一个独特事实，多轮后询问该事实
4. **预期**: 角色仍能引用（摘要生效）；压缩期间发消息无卡顿
5. 断网情况下继续聊到触发压缩 → 对话不中断（降级截断）

### V5 — 图片生成（US5）

1. 配置 Imagine Provider；角色开启「允许发送图片」
2. 发送明显求图消息（如「发张你的自拍」）
3. **预期**: 文字回复正常流式；图片随后出现在对话流中；重启后图片仍可见
   （已落盘 `Application Support/LoveChat/Images/`）
4. 删掉 Imagine Provider 的有效密钥再触发 → 文字回复完整，仅图片处提示失败

### V6 — 思考模式（US6）

1. Provider 开启思考模式，分别用支持/不支持的模型对话
2. **预期**: 两者均正常回复，不支持的模型无任何错误提示（静默降级）

### V7 — 错误路径（Edge Cases）

- 错误密钥 → 对话流中友好文案 + 重试按钮
- 流式中途断网 → 已收文字保留，可重新生成
- 删除正在使用的角色/Provider → 历史可浏览导出，继续发消息有明确提示
