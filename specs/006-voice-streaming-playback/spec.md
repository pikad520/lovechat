# Feature Specification: 外接语音分句流式播放开关

**Feature Branch**: `006-voice-streaming-playback`

**Created**: 2026-06-13

**Status**: Draft

**Input**: User description: "GPT-SoVITS 等外接语音引擎较慢，长回复要等整段合成完才出声。增加一个开关，在外接服务下选择：分句即时播放（首句合成完即播，后续边播边合成）或整段合成后一次性播放。"

## User Scenarios & Testing

### User Story 1 - 分句流式播放 (Priority: P1)

用户在语音设置开启「分句流式播放」后，绑定外接服务的角色朗读长回复时，第一句合成完
即开始播放，后续句子在后台继续合成并衔接，无需等整段完成。

**Acceptance Scenarios**:

1. **Given** 开关开启、角色绑定外接服务，**When** 朗读多句长回复，**Then** 首句就绪即出声（首声延迟≈一句的合成时间，而非整段）。
2. **Given** 正在分句播放，**When** 点停止或切换到其他消息，**Then** 立即停止合成与播放。
3. **Given** 某句合成失败，**When** 已有 Kokoro 模型，**Then** 该句回退内置引擎继续；否则停止并提示，不影响文字。

### User Story 2 - 一次性播放（开关关闭） (Priority: P2)

关闭开关时维持原行为：整段合成完一次性播放。

**Acceptance Scenarios**:

1. **Given** 开关关闭，**When** 朗读，**Then** 行为与本迭代前完全一致（整段合成后播放）。
2. **Given** 使用内置 Kokoro 的角色，**When** 朗读，**Then** 不受该开关影响（始终整段播放）。

### Edge Cases

- 极短回复（单句）→ 分句与整段无差异。
- 切句不破坏语义：按句末标点（。！？…；\n）切分，过短片段并入相邻句。

## Requirements

- **FR-501**: 提供「分句流式播放」开关，仅作用于外接语音服务；内置引擎始终整段播放。
- **FR-502**: 开启时按句切分，逐句合成，首句就绪即播，后续句后台预取并顺序衔接。
- **FR-503**: 分句路径下单句失败按既有规则降级（回退内置 / 提示），不影响文字对话；停止能即时中断合成与播放。
- **FR-504**: 同一时刻仅一路播放；切句不产生明显语义断裂。

## Success Criteria

- **SC-501**: 慢引擎（RTF≈1）下长回复首声延迟从「整段耗时」降到「首句耗时」。
- **SC-502**: 关闭开关时与迭代前逐字节一致（零回归）。
