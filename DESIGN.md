# LoveChat — 设计与需求文档

带界面的 AI 角色对话 macOS 应用。本文档为唯一需求基准,后续开发以此为准。

## 1. 平台与技术选型

- 目标平台:macOS 26 (Tahoe) 及以上,仅打包为 Mac App
- 技术栈:SwiftUI + SwiftData(本地持久化)+ 原生 URLSession 实现 SSE 流式,零第三方依赖
- 编译方式:生成完整 Xcode 工程,由用户在 Mac 上用 Xcode 打开编译运行(开发环境无法运行 xcodebuild)
- API Key 存储:macOS Keychain,严禁明文落盘

## 2. Provider 体系

### 2.1 AI Chat Provider(可配置多个)
- 支持两种协议:OpenAI 标准 API、Anthropic 标准 API
- 自定义字段:名称、Base URL、API Key、模型名、协议类型
- 聊天对话强制 stream(SSE);以下两类内部调用使用非流式:
  - "是否生成图片"判断调用(结构化结果,限制 max_tokens)
  - context 压缩调用
- 思考模式开关:
  - Anthropic → extended thinking(thinking budget)
  - OpenAI → reasoning effort
  - 模型不支持时静默降级(忽略参数),不报错、不中断对话
- 提供连通性测试功能

### 2.2 AI Imagine Provider(可配置多个)
- 优先支持 OpenAI 标准 Images API
- 自定义字段:名称、Base URL、API Key、模型名
- 生成的图片立即下载到本地存储(返回的 URL 短期过期,不可直接保存 URL)

## 3. 角色卡片

可创建多个角色,字段:名称、性别、外貌、灵魂(性格内核)、对用户的称呼、说话风格,及其他自由补充设定。

角色头像:
- 支持两种来源:本地上传图片,或调用 Imagine Provider 基于外貌设定生成
- 头像文件存本地;对话界面中角色消息旁持续显示头像;可更换、可移除

图片生成风格(角色级):
- 配置了 Imagine Provider 时,角色设定中需选择图片生成风格,同时作用于头像生成
  与对话中的情境生图
- 仅支持预置风格:3D写实 / 3D日漫 / 2D日漫(风格提示词写死在代码中,默认 3D写实)

角色级开关:
- 是否展示心理活动:提示词写死在代码中,角色不可自定义;心理活动用小括号 `(...)` 包裹。UI 解析括号内容渲染为特殊样式;模型不保证 100% 遵守格式,解析失败时按普通文本容错显示
- 是否允许发送图片:开启后,每条用户消息先额外调用一次 Chat Provider(非流式、短 prompt、限制 max_tokens)判断当前是否应该生图;判断为是则调用 Imagine Provider 生成一张图发给用户。生图提示词模板写死在代码中

## 4. 上下文与记忆

- 记忆对话轮数:可自定义,范围 1–20,默认 10
- context 压缩:对话超过 N 轮时,把滑出记忆窗口的旧消息用同一个 Chat Provider 压缩成摘要,注入 system prompt
  - 约束:压缩阈值 ≤ 记忆轮数(UI 层强制)
  - 压缩在后台异步执行,不阻塞用户发消息;压缩失败时降级为直接截断旧消息
- system prompt 组成 = 角色设定 + 心理活动提示词(若开启)+ 压缩后的历史摘要(若有)

## 5. 对话管理

- 旁白输入:聊天输入框上方有一个非必填的旁白输入框,用于描述场景/情节(非用户发言)。
  填写后与用户消息一起发送,提示词中明确区分旁白与用户发言;旁白在对话流中以
  区别于正文的样式展示,并随消息持久化
- 对话历史本地缓存(SwiftData)
- 卡片式历史对话列表,可选择任意历史对话继续;打开历史对话时自动定位到最新消息
- 可删除对话、可新开对话
- 对话内功能:停止生成、重新生成最后一条回复、消息编辑、消息删除、对话导出(Markdown)

## 6. 语音朗读(本地 TTS)

- 引擎:sherpa-onnx 推理库(vendored) + Kokoro 多语言 ONNX 模型(含中文音色),
  完全本地推理,无网络调用、无后台服务进程
- 设置中提供「语音」开关;开启时自动检测模型是否就绪,缺失则提供一键下载按钮
  (带进度展示),模型下载到本地数据目录,下载完成即可使用
- 模型不随安装包分发(控制安装包体积);仅在用户显式触发时下载
- 音色:提供中文预置音色选择(女声/男声若干),角色级可指定音色
- 交互:角色消息旁提供 🔊 播放按钮;角色级「自动朗读」开关,开启后每条回复
  完成时自动朗读;朗读心理活动括号内容时跳过
- 降级:模型缺失/推理失败时播放按钮置灰或轻量提示,绝不影响文字对话

外接语音服务(进阶,可选):
- 除内置引擎外,支持配置外部 TTS 服务,二选一协议:OpenAI 兼容
  (/v1/audio/speech)或 GPT-SoVITS(api_v2 /tts,支持参考音频克隆音色)
- 角色级可选择使用内置音色或某个外接服务的音色参数
- 外接服务不可达时自动回退内置引擎(若已就绪),否则按降级规则处理
- 外接服务可选「分句流式播放」开关:开启时长回复按句逐段合成、首句就绪即播,
  后续句后台预取(改善慢引擎首声延迟);关闭则整段合成完一次性播放。仅作用于
  外接服务,内置引擎不受影响

## 7. 桌面体验

- 侧栏左下角提供设置按钮(兼容 Windows 用户操作习惯,等价于 ⌘, 打开设置)
- 菜单栏 Help 提供应用内使用指引(目前仅中文)
- About 面板展示作者名称:pikad
- 应用菜单隐藏「服务」子菜单

## 8. 错误处理

- Provider 内容审核拒绝、网络错误、限流等 API 错误以友好文案展示在对话流中,可重试
- 流式中断可恢复或重新生成
- 生图失败不影响文字回复的正常展示

## 9. 写死在代码中的提示词(不可由用户/角色自定义)

1. 心理活动提示词:要求角色在回复中用小括号穿插心理活动
2. 生图判断提示词:根据最近对话判断当前是否应该发送图片,输出结构化结果
3. 生图提示词模板:基于角色外貌设定 + 当前对话情境生成图片描述
